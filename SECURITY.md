# Security

This project deals with presigned upload URLs, object storage credentials, and user-scoped upload sessions.

## Reporting A Vulnerability

Please do not open a public issue for a security vulnerability. Report it privately to the maintainer.

If you fork this project, replace this section with your own security contact.

## Security Assumptions

- S3 credentials are server-side only.
- Users are authenticated before creating uploads, checking status, signing parts, or completing uploads.
- Upload sessions are always scoped by user id.
- Object keys are generated by the server.
- File size and MIME type are validated on the server.
- The client cannot choose arbitrary bucket names.

## Common Misconfigurations

- Exposing S3 access keys in browser code.
- Allowing unauthenticated users to request signed part URLs.
- Trusting client-provided bucket names or object keys.
- Forgetting bucket CORS for browser `PUT` requests.
- Keeping abandoned multipart uploads forever.
